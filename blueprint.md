# Blueprint: 딩동댕닷컴

## 1. 프로젝트 개요

'딩동댕닷컴'은 사용자들이 다양한 주제의 미래 예측 퀴즈에 참여하고, 예측 성공 시 포인트를 획득하는 즐거움을 제공하는 웹 애플리케이션입니다. 투기나 도박이 아닌, 집단지성의 힘과 지적 유희를 핵심 가치로 삼습니다.

- **핵심 기능**: 퀴즈 참여, 포인트 시스템, 사용자 인증 및 랭킹
- **기술 스택**: HTML, CSS(TailwindCSS), JavaScript, Firebase (BaaS)
- **디자인 원칙**: 모던하고 직관적인 UI/UX, 다크 모드 지원, 완벽한 반응형 디자인

---

## 2. 현재 애플리케이션 상태 (v0.1)

- **UI/UX**:
    - 다크/라이트 모드 전환 기능 구현
    - TailwindCSS를 사용한 반응형 레이아웃 구성
    - 퀴즈 목록을 보여주는 아코디언 UI 구현
- **기능**:
    - 카테고리별 퀴즈 필터링 기능
    - 로그인/회원가입 모달 팝업 UI 및 뷰 전환 기능
    - `min-height`를 적용하여 팝업 높이 문제 해결

---

## 3. 새로운 계획: Firebase 백엔드 구축

**목표**: 사용자의 데이터를 안전하게 저장하고 관리하며, 포인트 시스템을 정교하게 운영하기 위한 Firebase 백엔드 환경을 구축합니다.

**작업 단계:**

1.  **[완료]** **Firebase 서버 환경 구성**: Firebase Studio 환경에 백엔드 서버를 사용하겠다고 선언합니다. (`.idx/mcp.json` 파일 생성)
2.  **Firebase SDK 및 설정 파일 추가**: 웹사이트가 Firebase와 통신할 수 있도록 `quiz.html`에 SDK를 추가하고, 개인화된 접속 정보를 담은 `firebase-config.js` 파일을 생성합니다.
3.  **회원가입 기능 구현**: 사용자가 회원가입 폼에 정보를 입력하고 버튼을 누르면, 이 정보가 Firebase Authentication (인증) 서비스와 Firestore (데이터베이스)에 안전하게 저장되도록 구현합니다.
    - **인증**: 사용자의 이메일/비밀번호를 사용하여 Firebase에 신규 계정을 생성합니다.
    - **데이터베이스**: 인증 성공 시, 해당 유저의 `uid`, `email`, `nickname`, `points` (기본값: 0), `createdAt` 등의 추가 정보를 Firestore 데이터베이스에 저장합니다.
4.  **로그인 기능 구현**: 사용자가 입력한 정보로 Firebase에 로그인을 요청하고, 성공 시 사용자 정보를 가져와 화면에 반영합니다.
5.  **포인트 시스템 설계**: 예측 결과에 따라 포인트를 지급/차감하는 로직을 안전하고 정확하게 처리할 수 있는 데이터 구조를 설계합니다.
