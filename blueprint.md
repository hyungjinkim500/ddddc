# Blueprint: 딩동댕닷컴

## 1. 프로젝트 개요

'딩동댕닷컴'은 사용자들이 다양한 주제의 미래 예측 퀴즈에 참여하고, 예측 성공 시 포인트를 획득하는 즐거움을 제공하는 웹 애플리케이션입니다. 투기나 도박이 아닌, 집단지성의 힘과 지적 유희를 핵심 가치로 삼습니다.

- **핵심 기능**: 퀴즈 참여, 포인트 시스템, 사용자 인증 및 랭킹
- **기술 스택**: HTML, CSS(TailwindCSS), JavaScript, Firebase (BaaS)
- **디자인 원칙**: 모던하고 직관적인 UI/UX, 다크 모드 지원, 완벽한 반응형 디자인

---

## 2. 현재 애플리케이션 상태 (v0.2)

- **UI/UX**:
    - 다크/라이트 모드 전환 기능 구현
    - TailwindCSS를 사용한 반응형 레이아웃 구성
    - 퀴즈 목록을 보여주는 아코디언 UI 구현
- **기능**:
    - 카테고리별 퀴즈 필터링 기능
    - Firebase Email/Password 기반 회원가입 및 로그인 기능 구현
    - 로그인/로그아웃 상태에 따른 UI 동적 변경 (버튼 토글)
    
---

## 3. 새로운 계획: 프로페셔널한 회원가입 경험 구축 (v0.3)

**목표**: 기존의 투박한 `alert` 기반 피드백을 제거하고, 사용자가 현재 어떤 상태인지 명확히 인지할 수 있는 세련된 UI/UX를 제공하여 회원가입 및 로그인 경험을 전문가 수준으로 향상시킵니다.

**작업 단계:**

1.  **`quiz.html` 구조 개선**:
    *   로그인/회원가입 모달 내부에 서버의 응답 메시지(성공/실패)를 표시할 전용 영역(`div`)을 추가합니다.
    *   로그인 후, 사용자 닉네임과 로그아웃 버튼을 표시할 개인화된 프로필 영역을 상단 네비게이션 바에 추가합니다.
    *   로그인/회원가입 버튼에 로딩 상태를 표시할 수 있는 스피너 아이콘과 텍스트를 담을 `span` 구조를 추가합니다.

2.  **`auth.js` 로직 강화**:
    *   회원가입 성공 시, Firebase Auth의 `updateProfile` 기능을 사용하여 사용자의 닉네임(`displayName`)을 저장합니다.
    *   `alert()`를 모두 제거하고, 새로 추가된 메시지 영역에 인증 과정에서 발생하는 모든 결과를 텍스트로 표시하도록 수정합니다.
    *   인증 요청 시, 버튼의 원래 텍스트를 숨기고 로딩 스피너 아이콘을 표시하여 비동기 작업이 진행 중임을 사용자에게 시각적으로 알립니다. 작업 완료 후에는 버튼을 원래 상태로 되돌립니다.

3.  **`quiz-main.js` 기능 확장**:
    *   사용자의 로그인 상태가 변경될 때(`onAuthStateChanged`), Firebase에 저장된 `displayName` (닉네임)을 가져와 상단 네비게이션 바의 프로필 영역에 표시합니다. 닉네임이 없다면 "사용자"와 같은 기본 이름을 표시합니다.
